6. **RecipeDetailPage** - Recipe display, forking, meal logging with rating & badges
7. **RewardsPage** - Achievements, badges (Waste Warrior, Budget Boss, Green Chef, Protein Pro), meal history
8. **ProfilePage** - User profile, dietary restrictions, allergies, health goals
9. **CommunityPage** - Browse & search community recipes with filters

### Features Implemented

- ✅ **JWT Auth Flow**: Register → Login → Store tokens → Auto-refresh on 401

- ✅ **Pantry-First Ordering**: UI shows perishable items with expiry warnings (red/yellow/green)
- ✅ **Safe-Filter Integration**: Backend filters recipes, frontend can highlight allergen conflicts
- ✅ **Meal Logging**: 5-star rating system with points & badge updates
- ✅ **Rewards Tracking**: Badge display, point accumulation, streak counter

## 🚀 How to Run

### Start Services (Manual)

```bash
# Terminal 1: Backend
cd Backend
python manage.py runserver

# Terminal 2: Frontend
cd Frontend
npm run dev
```

### Start Services (Automated - Windows)

```bash
double-click start-dev.bat
```

### Access Application

- Frontend: http://localhost:5173
- Backend API: http://localhost:8000/api/v1/
- API Docs: http://localhost:8000/api/docs/

## ⚙️ Configuration

### Frontend Config (vite.config.js)

```javascript
proxy: {
  '/api': {
    target: 'http://localhost:8000',
    changeOrigin: true,
  }
}
```

### Backend Configuration

Ensure you have `.env` file in Backend/ with:

- `OPENROUTER_API_KEY` - For recipe generation
- `PAYPAL_CLIENT_ID`, `PAYPAL_CLIENT_SECRET` - For payments
- `DEBUG=True` - For development
